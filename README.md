# Smart Table

Проект представляет собой реализацию интерактивной таблицы с данными о продажах. Таблица поддерживает сортировку, фильтрацию, поиск и пагинацию.

Основная задача проекта — показать подход к построению **модульной архитектуры на JavaScript** и продемонстрировать, как можно разбивать сложную функциональность на независимые части.

Проект собирается с помощью **Vite** и написан на **Vanilla JavaScript**.

---

## Функциональность

В таблице реализованы следующие возможности:

* отображение списка продаж
* поиск по данным
* фильтрация по параметрам
* сортировка по столбцам
* постраничная навигация
* динамическое обновление таблицы при изменении формы

Перерисовка таблицы происходит при событиях:

* `change`
* `submit`
* `reset`

Все действия пользователя приводят к повторной обработке данных и обновлению строк таблицы.

---

## Общая логика работы

Данные обрабатываются последовательно. Каждый этап изменяет текущий набор данных и передаёт результат следующему модулю.

Последовательность обработки выглядит так:

```
Исходные данные
→ поиск
→ фильтрация
→ сортировка
→ пагинация
→ вывод строк таблицы
```

Такой подход позволяет изолировать разные части логики и упрощает поддержку кода.

## Структура проекта

```
src
│
├── components
│   ├── table.js

│   ├── sorting.js
│   └── filtering.js
│
├── lib
│   └── вспомогательные функции
│

```

### components

В этой директории находятся основные модули таблицы:

* **table.js** — отвечает за вывод строк таблицы
* **pagination.js** — реализует постраничную навигацию
* **sorting.js** — сортировка данных
* **filtering.js** — фильтрация данных

### lib

Содержит вспомогательные функции, используемые в разных частях проекта.

### main.js

Главный файл приложения. Здесь происходит:

* инициализация модулей
* сбор состояния формы
* последовательное применение модулей обработки данных

---

## Принцип работы модулей

Каждый модуль инициализируется функцией, которая принимает настройки и возвращает функцию обработки данных.

Пример структуры модуля:

```javascript
function initModule(settings) {
  return (data, state, action) => {
    // обработка данных
    return data;
  };
}
```

Параметры:

* **data** — текущий набор данных
* **state** — значения элементов формы
* **action** — кнопка, вызвавшая действие (если есть)

---

## Применение модулей

В `main.js` данные последовательно проходят через все модули:

```javascript
result = applyFiltering(result, state, action);
result = applySorting(result, state, action);
result = applyPagination(result, state, action);
```

Каждый модуль изменяет данные и передаёт результат следующему.

---

## Пагинация

Модуль пагинации:

* вычисляет количество страниц
* показывает только строки текущей страницы
* обновляет кнопки навигации

Количество страниц определяется так:

```javascript
const pageCount = Math.ceil(data.length / rowsPerPage);
```

Для получения строк текущей страницы используется `slice`:

```javascript
const skip = (page - 1) * rowsPerPage;
data.slice(skip, skip + rowsPerPage);
```

---

## Сортировка

Сортировка выполняется по выбранному столбцу.

Состояние сортировки хранится в `dataset` кнопок:

```
data-field
data-value
```

`data-field` — поле данных
`data-value` — направление сортировки

Направление сортировки переключается по кругу:

```
none → asc → desc → none
```

---

## Фильтрация

Фильтрация выполняется на основе значений формы.

Пользователь может:

* выбрать продавца
* задать дополнительные параметры поиска
* использовать текстовый поиск

Все значения формы собираются в объект **state**, который используется при обработке данных.

---

## События

Таблица использует один механизм обработки событий.

Контейнер таблицы слушает события:

```
change
submit
reset
```

При любом из этих событий вызывается функция обновления таблицы, которая заново запускает обработку данных.

## Установка и запуск

Склонировать репозиторий:

```
git clone <repo-url>
```

Перейти в папку проекта:

```
cd project
```

Установить зависимости:

```
npm install
```

Запустить проект в режиме разработки:

```
npm run dev
```

После запуска в консоли появится адрес локального сервера, который нужно открыть в браузере.
